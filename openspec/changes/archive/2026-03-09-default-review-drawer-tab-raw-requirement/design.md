## Context

评审抽屉目前默认展示“评审报告”Tab，用户查看原始需求需要额外点击一次。结合当前使用场景，产品经理在进入抽屉后通常优先核对原始需求信息，再决定是否发起评审或查看报告。因此本次需要调整 Tab 初始激活项为“原始需求”，并保持其他交互行为不变。

## Goals / Non-Goals

**Goals:**
- 将 `DetailDrawer` 默认 Tab 从“评审报告”改为“原始需求”。
- 保持 Tab 切换、编辑、示例兜底等既有行为不变。
- 保持抽屉主操作按钮策略不变，仅调整默认首屏内容。

**Non-Goals:**
- 不改动 AI 评审生成逻辑。
- 不改动原始需求编辑/同步到 Meego 的流程。
- 不改动抽屉底部操作按钮文案与主次逻辑。

## Decisions

- 决策 1：通过修改 Tab state 的初始化值实现默认落点变更。
  - 理由：改动范围最小，风险低。
  - 备选：在打开抽屉时根据条件动态切换。未采用，因会增加状态分支复杂度。

- 决策 2：保持原始需求为空时 PRD 摘录兜底继续生效。
  - 理由：默认切到原始需求后更容易触发空态，必须确保首屏可读。
  - 备选：为空时自动切回评审报告。未采用，因违背“默认原始需求”目标。

## Risks / Trade-offs

- [风险] 用户原有习惯是先看评审报告。→ 缓解：保留一键切换到“评审报告”，不移除任何入口。
- [风险] 默认原始需求时，未评审卡片与已评审卡片首屏一致，可能降低评审结果曝光。→ 缓解：抽屉底部主按钮仍按阶段引导，保持行为提示。

## Migration Plan

1. 调整 `DetailDrawer` Tab 初始 state 为原始需求。
2. 回归验证：打开抽屉默认落在原始需求；手动切换评审报告正常。
3. 验证原始需求为空时示例兜底展示与标识正常。
4. 执行构建检查并记录结果。
