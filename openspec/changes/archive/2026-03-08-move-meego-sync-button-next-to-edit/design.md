## Context

在看板详情抽屉的“原始需求”Tab 中，“✎ 编辑”按钮位于标题行，而“同步到 Meego”按钮位于正文底部。两者同属高频操作却分散在不同区域，用户常在编辑完成后错过同步动作。该调整属于交互布局优化，目标是让关键动作更聚合。

## Goals / Non-Goals

**Goals:**
- 将“同步到 Meego”按钮移动到“编辑”按钮旁边，形成同一操作组。
- 保留同步逻辑（配置校验、请求、加载态、成功/失败提示）不变。
- 在编辑态与非编辑态切换时保持操作区布局稳定。

**Non-Goals:**
- 不改动 Meego API 协议、字段映射和请求参数。
- 不改动原始需求正文展示逻辑。
- 不重构详情抽屉整体布局。

## Decisions

- 决策 1：将同步按钮放入“原始需求（Meego）”标题行的右侧操作区，与“编辑”并列。
  - 理由：最接近用户心智路径（查看/编辑后立即同步）。
  - 备选：保留底部并新增吸顶按钮。未采用，信息冗余。

- 决策 2：非编辑态显示“编辑 + 同步到 Meego”；编辑态仅显示“保存/取消”。
  - 理由：避免编辑中触发同步导致状态冲突。
  - 备选：编辑态也显示同步按钮。未采用，易误触并增加认知负担。

- 决策 3：按钮样式沿用现有设计 token，仅调整位置与间距。
  - 理由：确保视觉一致与最小改动。
  - 备选：同步按钮改为图标按钮。未采用，影响可识别性。

## Risks / Trade-offs

- [风险] 标题行操作按钮变多后在小宽度下拥挤。→ 缓解：保持紧凑间距并允许换行前优先保证可点击。
- [风险] 习惯底部按钮的用户短期内需要适应。→ 缓解：文案不变、功能不变，仅位置调整。
- [权衡] 聚合操作提升效率，但减少了底部“流程收口”感。→ 缓解：保留现有保存/取消底部编辑操作组。

## Migration Plan

1. 调整 DetailDrawer 原始需求 Tab 操作区布局。
2. 删除底部同步按钮渲染分支。
3. 回归验证同步流程与编辑流程。
