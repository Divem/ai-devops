## Context

SDD2 设计工作台位于 `pm-ai-platform_sdd2.jsx`，是产品经理使用的主要工作界面。右侧内容面板顶部显示当前选中的需求 ID 和标题作为副标题（第 1118-1119 行）。当前实现为纯文本展示，无交互功能。

## Goals / Non-Goals

**Goals:**
- 在需求数据结构中添加飞书文档链接字段
- 将副标题中的需求名称改为可点击链接
- 点击后在新标签页打开飞书文档

**Non-Goals:**
- 不修改其他页面的链接显示逻辑
- 不实现飞书 API 集成（仅存储 URL）
- 不修改需求列表卡片的显示样式

## Decisions

- **字段设计**: 在需求卡片对象中添加可选的 `feishuUrl` 字段，类型为字符串，仅在有值时渲染链接
- **渲染策略**: 使用三元表达式判断，有 `feishuUrl` 时渲染 `<a>` 标签并设置 `target="_blank"`，否则渲染普通 `<span>`
- **样式处理**: 链接使用蓝色文字 + 下划线，与普通文字区分，当前文字颜色为 `C.muted`

## Risks / Trade-offs

- [风险] 飞书链接可能失效 → 链接在新标签页打开，用户可在原页面继续操作
- [风险] 历史需求数据无飞书链接 → 字段为可选，不影响现有功能
