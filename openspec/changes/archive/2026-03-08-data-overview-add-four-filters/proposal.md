## Why

当前数据概览页面的筛选入口偏“分析维度切片”，缺少与看板一致的业务筛选控件，用户无法直接按空间、子系统、应用、迭代快速聚焦数据，导致统计结果解释成本高、演示路径不连贯。

## What Changes

- 在数据概览页新增四个业务筛选条件：空间、子系统、应用、迭代
- 四个筛选条件与看板筛选规则保持一致，支持多选与联动（子系统→应用）
- 概览页面中的指标、趋势、漏斗、风险清单统一按筛选结果实时更新
- 增加筛选空态与“清除筛选”行为，确保交互可恢复
- 保持现有下钻行为，筛选条件可与下钻条件叠加生效

## Capabilities

### New Capabilities

（无）

### Modified Capabilities

- `data-overview-insights`: 扩展数据概览筛选能力，支持空间/子系统/应用/迭代四维业务筛选与联动更新

## Impact

- 主要影响：`pm-ai-app/src/PMPlatform.jsx` 数据概览区域与筛选状态管理
- 交互影响：数据概览顶部新增筛选条，和看板筛选语义一致
- 数据影响：继续基于现有 `cards` 前端计算，不新增后端 API
- 兼容性影响：保留现有“维度切片分析”和下钻能力，不破坏已实现功能
