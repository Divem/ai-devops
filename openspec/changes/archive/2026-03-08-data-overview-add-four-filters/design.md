## Context

数据概览页已经具备指标、趋势、漏斗与风险能力，但筛选方式仍偏“分析维度”，与看板页不一致。用户在演示或分析时，常先在看板里按空间/子系统/应用/迭代筛选，再切回概览查看统计，当前需要重复操作，效率较低。

现有约束：
- 当前筛选状态已由本地 state + localStorage 持久化
- 看板已支持空间、子系统、应用、迭代四类筛选与联动
- 数据概览所有统计都由前端 `cards` 派生计算

## Goals / Non-Goals

**Goals:**
- 在数据概览页提供与看板一致的四维筛选控件
- 筛选变化后实时驱动概览统计模块统一刷新
- 支持多选、联动和清除操作
- 保持下钻能力不受影响，支持与下钻条件叠加

**Non-Goals:**
- 不新增后端查询接口
- 不新增跨页面全局状态管理库
- 不改动卡片基础数据结构

## Decisions

1. 复用现有筛选状态，不新增第二套 state
- 选择：直接复用 `selectedSpaces/selectedSubsystems/selectedApps/selectedIterations`
- 原因：避免语义分裂，保证看板与概览筛选一致
- 备选：为概览单独维护筛选 state（放弃，维护成本高且易不一致）

2. 概览页顶部引入轻量筛选条
- 选择：在数据概览页顶部增加四个筛选入口和清除按钮
- 原因：用户心智一致，切换成本低
- 备选：侧边抽屉筛选（放弃，操作路径更长）

3. 应用筛选沿用子系统联动规则
- 选择：当子系统变化时，自动裁剪不合法应用选择
- 原因：避免无效筛选组合导致“空数据误判”
- 备选：允许任意组合（放弃，结果解释困难）

4. 空态策略统一
- 选择：筛选后无数据时，指标显示空态值并展示提示区块
- 原因：避免用户误以为统计计算错误
- 备选：仅保留图表空白（放弃，反馈不足）

## Risks / Trade-offs

- [Risk] 概览筛选控件与看板控件行为细节不一致 → Mitigation: 共用同一套 toggle/clear handler
- [Risk] 多维筛选叠加后频繁重算导致性能下降 → Mitigation: 继续使用 `useMemo` 并限制趋势点位数量
- [Risk] 下钻条件与手动筛选叠加后可读性下降 → Mitigation: 在筛选区展示下钻标签并支持单独清除

## Migration Plan

1. 在数据概览页接入四维筛选控件（空间/子系统/应用/迭代）
2. 将概览统计数据源统一切换为筛选后的 `filteredCards`
3. 复用现有筛选联动规则，确保子系统与应用一致性
4. 增加概览空态提示和清除筛选入口
5. 手工验证：多选、联动、清除、下钻叠加、刷新恢复
