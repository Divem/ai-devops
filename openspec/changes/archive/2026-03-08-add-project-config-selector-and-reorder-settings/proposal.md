## Why

当前项目配置页面缺少项目级关联信息（空间、子系统、应用）的集中配置，用户在提交文档或执行协作流程前无法先明确上下文归属。同时，当前配置区块顺序与使用心智不一致，降低了首次配置效率。

## What Changes

- 在项目配置页面新增"项目配置"区块并置顶，支持选择要关联的空间、子系统、应用。
- 将原有配置区块顺序调整为：项目配置 > Git 配置 > AI 模型配置（其余区块顺延保持不变）。
- 新增项目关联配置的持久化与回填能力，刷新后可恢复用户已选值。

## Capabilities

### New Capabilities
- `project-association-config`: 管理项目关联上下文（空间、子系统、应用）的选择、保存与恢复。

### Modified Capabilities
- `project-settings-page`: 更新项目配置页区块展示顺序，确保项目配置置顶且 Git 配置在 AI 模型配置之前。

## Impact

- 受影响代码：`pm-ai-app/src/PMPlatform.jsx`（项目配置页导航与内容渲染逻辑、配置保存逻辑）。
- 受影响数据：`projectConfig` 本地持久化结构新增项目关联字段。
- 对外 API：无新增后端依赖，默认先以本地配置与可扩展数据源方式实现。
