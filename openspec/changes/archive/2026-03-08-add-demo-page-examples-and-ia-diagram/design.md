## Context

`NEW_USER_DEMO_MANUAL.md` 与 `NEW_USER_DEMO_MANUAL_PPT.html` 是新用户理解平台功能的关键材料，但当前内容以文字为主，缺少典型页面视觉样例与系统结构总览图。为了让首次接触用户更快理解页面职责和模块关系，需要在文档与演示页中补齐页面示例和信息架构图，并保持二者叙事一致。

## Goals / Non-Goals

**Goals:**
- 为演示文档补充典型页面示例（如看板、详情、项目配置等）与预期页面效果说明。
- 在文档和 PPT 中增加统一的信息架构图，呈现核心模块层级与主要用户路径。
- 统一两份材料的章节顺序和术语，避免讲解口径不一致。

**Non-Goals:**
- 不改动业务代码逻辑与页面功能实现。
- 不引入新的设计系统或图表库依赖。
- 不扩展到完整视觉重设计，仅补充示例与结构图。

## Decisions

- 决策 1：页面示例采用“页面名称 + 关键区域 + 用户价值 + 讲解要点”结构。
  - 理由：既能描述视觉效果，又能直接用于演示讲解。
  - 备选：仅放截图。未采用，因为缺少语义说明，复用性低。

- 决策 2：信息架构图使用文本可维护格式（Mermaid 或等价结构化图示）嵌入文档与 PPT。
  - 理由：便于版本管理和后续修改。
  - 备选：仅放静态图片。未采用，因为更新成本高且不易 diff。

- 决策 3：先定义统一 IA 节点清单，再同步到两份材料。
  - 理由：可避免 md 与 html 内容漂移。
  - 备选：分别独立编写。未采用，因为后期维护容易产生不一致。

## Risks / Trade-offs

- [风险] 页面示例描述与实际 UI 版本可能存在时间差。→ 缓解：示例中标注“示意”并聚焦结构与流程。
- [风险] IA 图过于复杂影响新用户理解。→ 缓解：控制层级在 2-3 层，突出主路径。
- [权衡] 采用文本图示可维护性高，但视觉表现不如精绘图。→ 缓解：通过颜色/分组增强可读性。

## Migration Plan

1. 先在 `NEW_USER_DEMO_MANUAL.md` 补齐页面示例与 IA 图草案。
2. 将同一内容结构映射到 `NEW_USER_DEMO_MANUAL_PPT.html` 对应演示页。
3. 对照检查两份材料章节与术语一致性。
