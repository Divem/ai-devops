## Context

新增深色/浅色切换后，AI 辅助设计页面在不同视图中采用了“顶部内联按钮 + 右上角浮动按钮”混合策略。当前浮动按钮在部分分辨率与布局下与顶部设置按钮发生遮挡，且按钮基线不统一，影响视觉一致性和点击可达性。

本次变更聚焦布局修复：保证主题切换按钮与周边操作按钮（特别是设置按钮）不重叠，并在多视图下保持对齐规则一致。

## Goals / Non-Goals

**Goals:**
- 让主题切换按钮与顶部工具区元素在同一视觉基线或统一锚点。
- 消除主题切换按钮遮挡设置按钮的问题。
- 在不同页面模式（看板、详情、设置、设计）下使用一致的放置策略。

**Non-Goals:**
- 不调整主题功能本身（切换逻辑、持久化逻辑保持不变）。
- 不改动业务交互流程。
- 不引入新的 UI 框架或布局库。

## Decisions

- 统一主题按钮位置策略：优先放入现有头部工具区，避免独立 fixed 浮层覆盖交互控件。
  - 理由：头部工具区已承载设置等全局动作，天然避免遮挡。
  - 备选：保留 fixed 按钮并动态避让；未采用，维护复杂且易漏边界场景。

- 若某页面必须使用浮动按钮，采用统一偏移与 z-index 规范，并为设置入口预留安全间距。
  - 理由：保证跨页面行为可预测。
  - 备选：各页面独立微调像素值；未采用，容易再次不一致。

- 保持按钮尺寸与行高与周边按钮一致，确保视觉对齐。
  - 理由：减少“漂浮感”和错位感。
  - 备选：保留当前按钮尺寸；未采用，无法解决基线不齐问题。

## Risks / Trade-offs

- [头部空间不足] 小屏下按钮拥挤 → 在窄宽度下收敛为图标文案缩写或次级位置。
- [跨视图遗漏] 某些页面仍使用旧布局 → 逐视图核对主题按钮渲染入口。
- [层级冲突] 与弹层 z-index 关系异常 → 明确主题按钮仅在头部容器层，不跨越弹窗层级。

## Migration Plan

1. 梳理各页面主题按钮入口并统一布局策略。
2. 调整按钮尺寸、对齐、间距与层级，移除造成遮挡的 fixed 方案。
3. 回归验证设置按钮可点击、主题切换可用及小屏布局稳定性。
4. 若出现回归，回滚到上一个稳定入口并逐视图补丁。
