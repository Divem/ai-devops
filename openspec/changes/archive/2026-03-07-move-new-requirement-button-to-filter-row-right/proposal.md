## Why

当前看板页的“新建需求”入口与筛选区域分离，用户在筛选后进行新增操作需要额外视觉跳转，影响操作连贯性与页面信息层级。将其下移并与筛选条件同一行右侧展示，可强化“先筛选、再操作”的使用路径，提升效率与可发现性。

## What Changes

- 调整看板页工具条布局，将“新建需求”按钮从当前位置移动到筛选行。
- 在筛选行中保持筛选控件在左、“新建需求”按钮在右的单行布局。
- 保持现有按钮文案、功能行为、点击反馈与权限逻辑不变，仅变更布局与视觉位置。
- 在窄屏场景下确保不遮挡、不重叠，必要时允许同一工具行内自适应换行。

## Capabilities

### New Capabilities
- `kanban-toolbar-layout`: 规范看板页面筛选区与主操作按钮的同排布局、对齐方式与响应式行为。

### Modified Capabilities
- 无

## Impact

- 受影响模块：看板页面头部/工具条区域的 JSX 结构与内联样式。
- 受影响文件：`pm-ai-app/src/PMPlatform.jsx`（以及可能的预览入口对应组件文件）。
- 外部 API 与数据结构：无变化。
- 依赖与构建：无新增依赖，无后端改动。
