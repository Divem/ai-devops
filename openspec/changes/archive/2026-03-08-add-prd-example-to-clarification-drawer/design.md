## Context

当前看板页面在触发需求澄清时，用户只能看到问题与输入控件，缺少可参考的“原始需求样例”，导致首次使用时不知道应提供何种粒度的信息。项目内已有 `PM_AI_Plagform_PRD.md` 可作为稳定示例来源，适合在抽屉中摘录片段进行引导。该能力需要与现有澄清流程并存，且不能影响用户输入与提交路径。

## Goals / Non-Goals

**Goals:**
- 在需求澄清抽屉中展示一段来自 `PM_AI_Plagform_PRD.md` 的示例摘录，帮助用户理解“原始需求”应如何表达。
- 明确示例仅为参考内容，不回填、不覆盖用户字段。
- 当示例源不可用时提供可理解的降级提示，避免空白区域。

**Non-Goals:**
- 不实现完整 Markdown 渲染器或全文浏览功能。
- 不新增上传/编辑示例源文件能力。
- 不改变 AI 评审与提交流程的业务规则。

## Decisions

- 决策 1：在抽屉内新增只读“示例摘录”区块，而非弹窗。
  - 原因：与当前澄清交互在同一上下文内，减少来回切换。
  - 备选：弹窗展示示例；未采用，因会增加操作层级并打断输入。

- 决策 2：采用固定片段摘录策略（预设起止段或标记段），而非运行时语义抽取。
  - 原因：实现稳定、可预测，避免复杂解析导致前端波动。
  - 备选：按关键词实时抽取；未采用，因规则复杂且可维护性低。

- 决策 3：示例区默认展开且可滚动，长度受限。
  - 原因：确保可见性，同时避免挤压主要表单区域。
  - 备选：默认折叠；未采用，因降低引导效果。

- 决策 4：示例加载失败时展示降级文案，不阻断主流程。
  - 原因：示例是增强能力，不应成为流程阻塞点。
  - 备选：报错并禁止继续；未采用，因与需求目标不符。

## Risks / Trade-offs

- [Risk] PRD 文件路径或命名变化导致示例无法读取 → Mitigation：提供稳定兜底文案并记录控制台告警。
- [Risk] 摘录过长影响抽屉可用性 → Mitigation：限制最大显示高度并启用内部滚动。
- [Risk] 用户误解示例为必填模板 → Mitigation：在区块标题与说明中明确“仅作示例参考”。
- [Trade-off] 固定摘录的灵活性较低，但换来实现简单与行为稳定。
