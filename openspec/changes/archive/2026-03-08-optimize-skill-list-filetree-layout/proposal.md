## Why

当前 AI Skill 页面中“Skill 列表”和“文件结构”视觉上分离，间距较大，导致用户在切换 Skill 后定位文件树需要额外视线跳转。为了提升配置效率与可读性，需要优化两者的布局关系，使其更贴合、连续且易于扫描。

## What Changes

- 调整 AI Skill 区块布局，将 Skill 列表与文件结构在视觉上贴近并形成一体化导航区域。
- 优化两列容器间距、边框和内边距，使“Skill -> 文件”的层级关系更明确。
- 在窄屏场景下提供等效紧凑布局，避免布局拥挤或错位。
- 保持现有编辑、保存、恢复默认等交互行为不变，仅优化信息架构与样式。

## Capabilities

### New Capabilities
- （无）

### Modified Capabilities
- `ai-skill-management`: 调整 Skill 列表与文件结构的布局和呈现方式，提升导航连续性。
- `project-settings-page`: AI Skill 区块在配置页中的样式与间距规范更新，保证顺序不变但视觉更紧凑。

## Impact

- 受影响代码：`pm-ai-app/src/PMPlatform.jsx`（AI Skill 板块布局与样式）。
- 受影响范围：前端展示层，无数据模型变更、无 API 变更。
- 风险：低，主要为 UI 调整，需回归验证桌面与窄屏显示效果。

## 验收要点

- Skill 列表与文件结构在同一导航组内连续展示，视觉上贴合。
- 桌面与窄屏场景下，编辑区不被遮挡，交互路径保持清晰。
- 保存、恢复默认、文件切换行为与改造前一致。
