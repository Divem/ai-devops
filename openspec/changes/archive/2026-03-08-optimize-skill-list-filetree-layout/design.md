## Context

AI Skill 管理区块已从单编辑器升级为 Skill 列表 + 文件结构 + 编辑器的组合模式，但当前 Skill 列表和文件结构在视觉上存在较大间距，阅读流不连续。用户在频繁切换 Skill 与文件时，需要在两块区域之间反复跳转视线，影响操作效率。本次变更仅针对布局与样式优化，不改动数据模型与业务逻辑。

## Goals / Non-Goals

**Goals:**
- 让 Skill 列表与文件结构视觉上“贴在一起”，形成统一导航区域。
- 优化左右分栏间距、边框和内边距，突出层级关系（Skill > 文件）。
- 保证窄屏下仍有可用的紧凑布局，不出现遮挡或错位。
- 保持保存、恢复默认、文件切换等现有交互行为不变。

**Non-Goals:**
- 不改动 Skill 配置数据结构与持久化逻辑。
- 不改动 AI prompt 生成或调用链路。
- 不新增后端接口或外部依赖。

## Decisions

- 决策 1：将 Skill 列表与文件结构放入同一容器组，并共享统一外框样式。
  - 理由：通过连续视觉块降低认知切换成本。
  - 备选：保持两个独立卡片，仅缩小 gap。未采用，因一体化感知不足。

- 决策 2：使用更小的水平 gap 与更紧凑的 padding。
  - 理由：满足“贴在一起”的用户诉求，同时保留可读性。
  - 备选：完全取消间距。未采用，因会导致内容拥挤、层级不清。

- 决策 3：窄屏下优先保持垂直堆叠紧凑顺序（Skill 列表紧邻文件结构）。
  - 理由：避免横向空间不足造成控件挤压。
  - 备选：隐藏文件树改抽屉。未采用，因实现成本与交互变化更大。

## Risks / Trade-offs

- [风险] 过度紧凑导致点击目标区域偏小。→ 缓解：保留最小按钮高度与 hover 高亮。
- [风险] 新样式与已有 sectionStyle 冲突。→ 缓解：仅在 AI Skill 区块局部覆盖样式，不影响其他区块。
- [权衡] 视觉一致性与“贴合”之间需要平衡。→ 缓解：统一外框+轻量间距，而非完全无间隔。

## Migration Plan

1. 调整 AI Skill 区块 JSX 容器结构（合并左两列为导航组）。
2. 调整样式参数（gap/padding/width/border）并保持交互逻辑不变。
3. 验证桌面与窄屏显示效果，确认保存与切换行为不受影响。
4. 若回滚，仅恢复旧布局样式，不涉及数据迁移。
