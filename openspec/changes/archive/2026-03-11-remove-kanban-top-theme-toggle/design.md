## Context

看板页顶部导航当前同时存在“独立主题切换按钮”和“设置下拉入口”，而设置下拉中已具备主题切换能力。重复入口会造成两类问题：

1. 用户不确定首选入口，增加操作路径不一致。
2. 顶部导航控件过密，影响主信息（导航标签、统计信息、设置入口）的可读性。

最近改动已将主题切换接入设置下拉，因此本次属于入口收敛与布局简化，不涉及主题逻辑本身。

## Goals / Non-Goals

**Goals:**
- 移除看板页顶部独立主题按钮，保留设置下拉中的主题切换为唯一入口。
- 确保主题切换效果和持久化行为与现状一致（即时生效、刷新恢复）。
- 保持看板顶部导航布局稳定，不影响设置下拉和其它操作。

**Non-Goals:**
- 不调整主题 token、配色细节或深浅色渲染逻辑。
- 不新增新的设置面板或快捷键入口。
- 不改动详情页、设计页和设置页已有主题实现（仅做一致性校验）。

## Decisions

1. **看板页只保留设置下拉入口，不再显示独立主题按钮**
   - 决策：删除看板顶部 `ThemeModeToggle` 渲染，主题能力仅通过 `SettingsDropdown` 菜单项触发。
   - 原因：减少重复入口，统一心智模型。
   - 备选：保留独立按钮并在设置下拉保留同功能；入口重复问题仍存在。

2. **复用现有主题状态与持久化链路**
   - 决策：继续使用 `themeMode` 状态、`setThemeMode` 切换逻辑和 localStorage 持久化，不引入新状态。
   - 原因：风险最低，改动最小。
   - 备选：将主题状态下沉到新上下文层；超出本次范围。

3. **保持设置下拉菜单项文案可识别当前模式**
   - 决策：菜单项显示“主题：浅色/深色”，点击即切换并关闭下拉。
   - 原因：替代独立按钮后，需要明确当前状态与下一步动作。
   - 备选：仅显示“主题切换”；状态信息不够直观。

## Risks / Trade-offs

- [风险] 用户短期内找不到主题入口 → [缓解] 在设置菜单项显式显示“主题：浅色/深色”。
- [风险] 顶部布局变动导致间距突兀 → [缓解] 回归检查导航统计块与设置按钮的间距。
- [风险] 多视图入口策略不一致 → [缓解] 联合回归看板/详情/设计/设置页，确保规则统一。

## Migration Plan

1. 删除看板顶部独立主题按钮渲染。
2. 确认设置下拉包含并可触发主题切换。
3. 回归主题即时生效、刷新恢复、设置页跳转与返回流程。
4. 若发现入口不可达，回滚为“仅恢复独立按钮渲染”临时兜底。
