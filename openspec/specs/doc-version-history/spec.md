## ADDED Requirements

### Requirement: AI 设计页显示版本记录区域
系统 MUST 在 AI 设计页面右侧面板中于 Chatbot/参考资料之后提供版本记录区域，用于展示当前需求文档的版本更新日志。

#### Scenario: 打开 AI 设计页面
- **WHEN** 用户进入任一需求的 AI 设计页面
- **THEN** 右侧面板可看到版本记录区域，且位置在 Chatbot 与参考资料之后

### Requirement: 版本记录展示核心版本信息
系统 SHALL 在版本记录中展示每次版本更新的核心信息，至少包含版本标识、更新时间、提交说明与关联文档范围。

#### Scenario: 查看已有版本日志
- **WHEN** 当前需求存在历史版本记录
- **THEN** 用户可按时间顺序查看版本列表并识别每个版本的关键信息

### Requirement: 支持提交为新版本并写入日志
系统 MUST 提供“提交为新版本”操作，提交到 GitHub 成功后 SHALL 新增一条版本记录并在界面即时可见。

#### Scenario: 提交当前文档为新版本成功
- **WHEN** 用户点击“提交为新版本”且 GitHub 提交成功
- **THEN** 系统在版本记录中新增最新版本条目并显示成功反馈

### Requirement: 提交失败不得写入虚假版本
系统 MUST 在 GitHub 提交失败时阻止新增版本记录，并向用户展示失败状态与原因提示。

#### Scenario: 提交当前文档为新版本失败
- **WHEN** 用户点击“提交为新版本”但 GitHub 返回错误
- **THEN** 版本记录不新增条目，且界面显示失败提示
