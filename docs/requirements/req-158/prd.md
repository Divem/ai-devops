## 1. 概述

### 1.1 背景与目标

**背景**: 当前系统缺乏统一的业务度量能力，用户无法直观掌握核心业务指标的运行状况。各部门数据分散在多个子系统中，数据分析师需要手动编写SQL查询并导出至Excel进行分析，效率低下且存在数据口径不一致的问题。管理层缺乏实时、可视化的业务健康度监控手段，关键业务异常无法及时发现，决策滞后于市场变化。随着业务规模扩大，数据驱动决策的需求日益迫切，亟需构建一套完整的度量体系，实现指标的自动化采集、智能计算、可视化展示及主动预警。

**目标**: 
- **短期目标（MVP）**：上线度量指标配置与基础可视化能力，支持5种核心度量类型（计数、求和、平均值、比率、自定义SQL），提供折线图、数字卡片两种展示形式，实现T+1小时的数据更新延迟
- **中期目标（Q2-Q3）**：完善度量看板自定义布局、多维度下钻分析、阈值预警通知机制，支持数据导出功能，数据更新延迟优化至配置频率的1.5倍以内
- **长期目标（Q4及以后）**：构建智能度量洞察引擎，支持异常自动归因分析、预测性指标推荐、自然语言查询度量数据，开放完整API生态，成为企业级统一指标中台

### 1.2 产品愿景

**"让每一个业务决策都有数据支撑，让每一次指标波动都被即时感知"** —— 打造企业级一站式度量平台，实现从数据定义到决策行动的全链路闭环。

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 度量指标管理（Metric Management）

| 功能模块 | 详细说明 | 优先级 |
|---------|---------|--------|
| **指标创建向导** | 四步引导式配置：①基础信息（名称、描述、业务分类、责任人）②数据源选择（支持单表/多表关联/自定义SQL/外部API）③计算逻辑（内置函数库+公式编辑器）④更新策略（实时/准实时/定时，最小粒度5分钟） | P0 |
| **度量类型引擎** | 支持五种核心类型：<br>• **计数型**：去重计数、累计计数<br>• **求和型**：简单求和、加权求和<br>• **平均值型**：算术平均、加权平均、移动平均<br>• **比率型**：转化率、占比、完成率（自动处理除零保护）<br>• **自定义SQL**：完整SQL编辑器，支持CTE、窗口函数、参数化查询 | P0 |
| **指标版本管理** | 每次修改自动保存历史版本，支持版本对比与一键回滚，保留最近50个版本 | P1 |
| **指标血缘追踪** | 可视化展示指标依赖的数据源、上游指标、下游看板，变更影响一键分析 | P1 |
| **指标目录** | 树形分类体系（最多5级），支持标签体系与全文检索，智能推荐相似指标 | P1 |

#### 3.1.2 可视化展示引擎（Visualization Engine）

| 功能模块 | 详细说明 | 优先级 |
|---------|---------|--------|
| **数字卡片** | 展示当前值、对比值（同比/环比）、趋势箭头、目标完成进度条；支持阈值颜色预警（绿/黄/红） | P0 |
| **折线图** | 支持单/多指标对比、双Y轴、数据点标记、区域填充、预测线延伸；交互：框选缩放、数据点下钻 | P0 |
| **柱状图** | 分组柱状图、堆叠柱状图、瀑布图模式；支持排序、TopN筛选、标签旋转 | P0 |
| **仪表盘** | 环形/半环形/全环形三种样式，自定义刻度区间与颜色分段，指针动画效果 | P1 |
| **组合图表** | 单画布混合多种图表类型，支持联动筛选器（时间/维度/指标） | P1 |
| **下钻分析** | 预设下钻路径（如：全国→大区→城市→门店），或自定义维度组合下钻 | P1 |

#### 3.1.3 度量看板（Metric Dashboard）

| 功能模块 | 详细说明 | 优先级 |
|---------|---------|--------|
| **画布布局系统** | 响应式网格布局（12列），拖拽调整组件位置与大小；预设模板（运营看板/财务看板/高管驾驶舱） | P0 |
| **组件库** | 图表组件、筛选器组件（日期/单选/多选/级联）、文本组件（标题/说明/指标解读）、媒体组件（图片/视频） | P0 |
| **交互配置** | 跨组件联动筛选、图表跳转（单击/双击/悬停触发）、自定义URL传参 | P1 |
| **看板权限** | 公开/私密/密码保护三种分享模式，支持嵌入第三方系统（iframe/JS SDK） | P1 |
| **移动端适配** | 自动适配手机/平板，支持手势操作，关键指标推送至企业微信/钉钉 | P1 |

#### 3.1.4 预警与通知（Alert & Notification）

| 功能模块 | 详细说明 | 优先级 |
|---------|---------|--------|
| **阈值规则引擎** | 静态阈值（绝对值/百分比）、动态阈值（基于历史数据的标准差/分位数）、复合条件（多指标组合判断） | P0 |
| **预警级别** | 提示（蓝色）/警告（黄色）/严重（红色）/紧急（紫色）四级，对应不同通知策略 | P0 |
| **通知渠道** | 站内信、邮件、短信、钉钉机器人、企业微信机器人、飞书机器人、Webhook自定义回调 | P0 |
| **通知内容** | 富文本模板编辑器，支持变量插入（指标值、阈值、触发时间、对比数据）、一键跳转看板链接 | P1 |
| **预警降噪** | 连续触发合并、静默期设置、预警升级策略（5分钟无响应自动升级通知对象） | P1 |
| **预警治理** | 预警准确率统计、误报反馈入口、智能推荐阈值调整建议 | P2 |
