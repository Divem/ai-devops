## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 多平台 Git 集成管理

| 功能项 | 详细说明 |
|:---|:---|
| 平台配置 | 支持 GitHub、GitLab、Gitee 三大代码托管平台的 OAuth/App 授权接入 |
| 仓库绑定 | 按项目维度配置目标仓库、目标分支及基础目录路径 |
| 凭证安全 | 采用加密存储访问令牌，支持 Token 自动轮转与过期预警 |

**关键业务规则**：
- 每个 OpenSpec 项目仅允许绑定一个主仓库，支持配置备用仓库
- 授权失效时自动阻断同步操作并触发通知
- 平台选择受企业安全策略管控，未授权平台不可见

#### 3.1.2 自动化 Git 提交

| 功能项 | 详细说明 |
|:---|:---|
| 触发机制 | 产品经理点击"确认并同步"按钮后自动触发，或配置自动同步策略 |
| 目录规范 | 强制提交至 `openspec/changes/<proposal-name>/` 目录结构 |
| 文件命名 | 自动生成规范文件名：`{YYYYMMDD}-{proposal-id}-{version}.md` |
| Commit 信息 | 预填充标准化提交信息，包含提案标题、作者、变更摘要 |

**关键业务规则**：
- 目录层级不可配置，确保全平台统一规范
- 同名文件自动递增版本号，禁止覆盖历史版本
- 单次提交文件大小上限 10MB，超大附件转存对象存储并记录引用

#### 3.1.3 可选 PR/MR 工作流

| 功能项 | 详细说明 |
|:---|:---|
| 模式选择 | 支持"直接提交"与"创建 PR/MR"两种模式，项目级配置默认策略 |
| 自动创建 | 选择 PR/MR 模式时，自动填充标题、描述、指派人（可配置规则） |
| 链接回写 | PR/MR 创建成功后，自动将链接回写至 OpenSpec 提案详情页 |
| 状态同步 | 监听 PR/MR 状态变更（Open/Merged/Closed），同步至提案生命周期 |

**关键业务规则**：
- PR/MR 描述自动关联提案原始链接，便于双向追溯
- 合并冲突时暂停自动处理，转人工介入并标记状态
- 回写链接 5 分钟内未完成则标记为"异步处理中"

#### 3.1.4 同步可靠性保障

| 功能项 | 详细说明 |
|:---|:---|
| 重试机制 | 网络异常时自动重试 3 次，指数退避策略（1s/2s/4s） |
| 成功率监控 | 实时统计 Git 提交成功率，目标 ≥99.5% |
| 失败告警 | 同步失败时通过站内信、邮件、Webhook 多渠道通知 |
| 死信队列 | 最终失败任务进入死信队列，支持运营后台手动重试或导出 |

### 3.2 扩展功能（可选）

### 4.1 性能要求

| 指标 | 要求 | 说明 |
|:---|:---|:---|
| 同步响应时间 | P99 < 3s | 从点击确认到收到"已提交"反馈 |
| PR/MR 创建耗时 | P99 < 8s | 含链接回写完整流程 |
| 并发处理能力 | ≥100 TPS | 保障大规模团队同时使用 |
| 队列积压处理 | 单队列积压 >500 时自动扩容 | 防止高峰期任务堆积 |

### 4.2 安全性要求

- **传输安全**：全链路 HTTPS/TLS 1.3 加密
- **凭证隔离**：各租户 Git 凭证独立存储，禁止跨租户访问
- **操作审计**：完整记录谁、何时、对哪个仓库执行了什么操作，保留 180 天
- **敏感信息扫描**：提交前自动检测 API Key、密码等敏感信息，阻断并告警

| 指标 | 要求 |
|:---|:---|
