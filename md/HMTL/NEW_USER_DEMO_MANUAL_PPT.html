PM AI Platform 用户手册

这是一份“可直接上手”的使用指南。目标是让用户拿着这份手册就能知道平台有什么功能、解决什么问题，以及如何从需求录入一路走到文档交付和数据复盘。

1. 平台介绍

PM AI Platform 是一个面向需求管理与方案产出的 AI 工作台。它把需求从“收集、评审、设计、文档化、复盘”放进同一条流程。

一句话：用看板管理进度，用 AI 提升质量，用文档沉淀结果，用数据做复盘。

2. 解决的问题

  • 需求信息不完整：AI 评审输出结构化问题与建议。
  • 评审效率低：统一评分视图，减少主观争议。
  • 文档产出慢：自动生成 PRD / Proposal / Design / Spec / Tasks。
  • 过程不可复盘：数据概览支持维度筛选和趋势观察。

3. 功能地图

需求评审看板

  • 6 阶段流转：待评审、评审中、AI 分析中、人工确认、已通过、已拒绝。
  • 四维筛选：空间、子系统、应用、迭代。
  • 新建需求入口在筛选行右侧。

需求详情页

  • 左：文档树（支持多提案文件夹）。
  • 中：文档预览与编辑。
  • 右：AI 对话与参考资料。

项目配置

  • AI 模型：Claude / GLM-4 / ARK Code Latest。
  • Git 仓库配置、SDD 框架模板、AI Skill 配置。

数据概览

  • 看总量、通过率、覆盖率、阶段分布。
  • 按阶段/优先级/空间/迭代下钻。

4. 典型页面示例

以下示例与演示手册一致，统一采用“页面名称 / 关键区域 / 用户价值 / 讲解要点”。

需求评审看板

关键区域：筛选条、六阶段看板列、需求卡片状态

用户价值：可视化推进需求，快速识别积压

讲解要点：演示卡片拖拽 + 按空间或迭代筛选

需求详情页

关键区域：文档树、编辑区、AI 对话与参考

用户价值：在同一上下文完成评审到文档产出

讲解要点：展示 PRD / Spec / Tasks 联动

项目配置页

关键区域：项目关联配置、Git 仓库配置、AI 模型配置

用户价值：统一项目上下文和能力配置

讲解要点：先项目关联，再 Git，再 AI

数据概览页

关键区域：指标卡、阶段分布、筛选下钻

用户价值：同时服务管理复盘与执行改进

讲解要点：切换筛选后对比关键指标变化

6. 快速上手

启动应用

cd pm-ai-app
npm install
npm run dev

访问 http://localhost:5173

配置 AI 模型

  1. 右上角 ⚙ → 项目配置 → AI 模型配置
  2. 选择 Claude / GLM / ARK
  3. 输入 Key 并保存
即使没有 Key，也可体验流程：系统会在失败时回退示例数据。

跑通一条需求

  1. 创建或选择需求卡片
  2. 执行 AI 智能评审并修订内容
  3. 生成文档并进入任务拆解

7. 推荐流程

  1. 需求录入（补全标题、描述、用户故事、验收标准）
  2. AI 初评（先发现风险与缺口）
  3. 人工确认（集中决策关键问题）
  4. 文档生成（PRD / Proposal / Design / Spec / Tasks）
  5. 任务排期（研发按 Tasks 执行）
  6. 数据复盘（识别滞留阶段与改进点）

8. 常用操作

文档树管理建议

  • 一个需求建议保留一个主 PRD。
  • 多提案文件夹用于并行方案对比。
  • Proposal / Design / Spec / Tasks 建议成组维护。

AI 对话提问模板

  • “请补全这条需求最容易遗漏的验收标准。”
  • “把当前风险拆成业务风险与技术风险。”
  • “将当前内容整理成可评审版本。”

9. 常见问题

Q1：提示未配置 API Key

到项目配置中确认模型与 Key 对应关系是否正确（Claude/GLM/ARK）。

Q2：模型切换后无变化

确认点击了保存配置，并刷新页面后重新触发 AI 请求。

Q3：AI 请求失败是否还能继续

可以，系统有回退示例数据，不影响主流程体验。

10. 新用户首周建议

  • Day 1：录入 5-10 条真实需求，统一字段规范。
  • Day 2：优先级高的需求执行 AI 评审并人工复核。
  • Day 3：生成 Spec + Tasks 并启动研发排期。
  • Day 4-5：沉淀评审模板和文档模板。
  • 周末：用数据概览做小复盘。

11. 使用目标

完成本手册后，用户应能：

  • 清晰说明平台功能与价值。
  • 独立跑通“需求到文档”的完整链路。
  • 正确配置并使用任意一个 AI 模型。
  • 在团队内落地“AI 初评 + 人工决策 + 数据复盘”的工作方式。
