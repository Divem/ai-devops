## 用户旅程对照（章节 2）

| 步骤 | PRD 描述 | 实现状态 |
|------|----------|----------|
| 1. 触发需求分析 | 点击"AI 辅助分析"按钮 | ✅ 点击看板卡片 → 评审抽屉；或直接进入 AI 设计页 |
| 2. 多轮对话澄清 | 与 AI 多轮对话，实时更新结构化草案 | ⚠️ Chatbot 存在，但对话不联动结构化字段；无主动提问 |
| 3. OpenSpec 提案生成 | 从对话结果一键生成 | ⚠️ 已实现生成，但入口为 PRD 页面的"生成提案"按钮，非对话触发 |
| 4. 提案在线评审 | 富文本编辑器 + 版本管理 | ⚠️ 有编辑功能，无版本管理 |
| 5. 确认并同步至 Git | 整体提案包提交，可选 PR | ⚠️ 单文档提交，无整体打包，无 PR |
| 6. 开发拉取执行 | Git 仓库侧行为 | — 超出前端范围 |
| 7. 评审与归档 | 平台归档提案 | ❌ 归档为 CLI 工具功能，平台内无归档操作入口 |

## 超出 PRD 的实现（当前版本额外特性）

以下功能在 PRD 中未提及，但已在当前版本中实现，提升了产品完整度：

| 功能 | 说明 |
|------|------|
| **AI Skill 管理** | 7 个内置 Skill，支持自定义 prompt 模板 + 一键恢复默认 |
| **多 AI 模型支持** | Claude (Anthropic) + GLM-4 (智谱)，项目配置页切换 |
| **历史需求参考面板** | 基于标签匹配的相似需求推荐（ReferencePanel），支持手动添加链接 |
| **AI 失败自动兜底** | AI 请求失败时自动注入示例数据，全流程可离线演示 |
| **三连击快速编辑** | 在 PRD/Proposal/Spec 预览区三连击直接进入编辑态 |
| **多提案支持** | 一个需求可生成多个 OpenSpec 提案（数组结构），文档树分文件夹展示 |
| **看板卡片 Badge 筛选** | 点击卡片上的维度 badge 一键触发对应筛选 |

## 优先补齐建议

按 PRD 核心用户旅程对齐度排序：

| 优先级 | 缺口 | 影响 |
|--------|------|------|
| P0 | **Meego 数据同步**（看板数据源） | 当前全靠 sample data，无法对接真实业务 |
| P0 | **AI 主动澄清 + 弹框确认** | PRD 核心交互，差距最明显 |
| P1 | **对话结构化输出联动**（对话 → 更新需求字段） | 用户旅程步骤 2 的关键产出 |
| P1 | **整体提案包提交 Git** | 当前单文档提交体验碎片化 |
| P2 | **版本历史** | 提案评审闭环必要条件 |
| P2 | **拖拽流转** | PRD 明确要求，当前为按钮替代 |
| P3 | **分屏对比 / 评论批注** | 协作评审能力，可后续迭代 |
| P3 | **PR/MR 自动创建** | PRD 标注可选 |
